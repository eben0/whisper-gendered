# Claude Code Build Prompt — Gender-Aware Hebrew Subtitle Server

## What to build

A standalone Python server that acts as a **Bazarr-compatible Whisper ASR provider**.
Bazarr will send it audio files and expect subtitle content back — exactly as it does
with `whisper-asr-webservice` or `subgen`. On top of standard transcription, this server
adds a full gender-aware English → Hebrew translation pipeline.

---

## Background context (read before writing any code)

### How Bazarr talks to a Whisper provider

Bazarr sends audio to a running HTTP server. The relevant endpoint is:

```
POST /asr
Content-Type: multipart/form-data

Fields:
  audio_file  — the audio/video file bytes
  task        — "transcribe" or "translate"
  language    — ISO 639-1 source language code (e.g. "en")
  output      — desired format: "srt" | "vtt" | "json" | "txt" | "tsv"
  encode      — "true" | "false" (whether to re-encode audio with ffmpeg first)
```

The response body must be the raw subtitle content in the requested format
(e.g. a plain `.srt` string for `output=srt`).

Bazarr also polls `GET /` or `GET /docs` to verify the server is alive, so the root
route should return a basic status JSON.

Reference implementation to study:
https://github.com/ahmetoner/whisper-asr-webservice

### The translation pipeline

When `task=transcribe` and the output language is Hebrew (configured server-side via
env var), the server must do more than plain transcription:

1. **Transcribe** audio → English text with word-level timestamps (faster-whisper)
2. **Diarize** speakers → who speaks when (pyannote.audio)
3. **Detect gender** per speaker → male / female, using median fundamental frequency
   (F0) via `librosa.pyin`. Males ~85–165 Hz, females ~165–255 Hz. Threshold: 165 Hz.
4. **Translate** each segment → Hebrew via Claude API, passing the speaker's gender
   in the system prompt so Hebrew grammar (verb conjugation, adjective agreement,
   imperatives) is correct for that speaker.
5. **Return** properly formatted SRT/VTT with Hebrew text.

When `TRANSLATE_TO_HEBREW=false`, the server behaves as a plain faster-whisper
transcription server (standard Whisper ASR webservice behaviour).

## Detailed spec per file

### `config.py`

Load all config from environment variables (use `python-dotenv` to read `.env`).
Expose a single `Settings` dataclass or similar. Variables:

| Env var              | Default         | Description                                      |
|----------------------|-----------------|--------------------------------------------------|
| `PORT`               | `9000`          | HTTP port to listen on                           |
| `WHISPER_MODEL`      | `large-v3`      | faster-whisper model name                        |
| `COMPUTE_TYPE`       | `float16`       | CTranslate2 quantisation                         |
| `DEVICE`             | `cuda`          | `cuda` or `cpu`                                  |
| `CONCURRENT_JOBS`    | `1`             | Max parallel transcription jobs                  |
| `TARGET_LANGUAGE`    | `none`          | Translate output to this language (e.g. `Hebrew`, `Arabic`, `French`). Set to `none` to disable translation and return plain transcription. |
| `HF_TOKEN`           | *(required)*    | HuggingFace token for pyannote                   |
| `ANTHROPIC_API_KEY`  | *(required)*    | Anthropic API key for Claude                     |
| `CLAUDE_MODEL`       | `claude-opus-4-6` | Claude model to use for translation            |
| `GENDER_THRESHOLD_HZ`| `165`           | F0 Hz boundary between male and female           |
| `DEBUG`              | `false`         | Verbose logging                                  |

**Gender-aware languages:** gender detection and gendered grammar prompting must be
enabled for languages where speaker gender affects grammar. The server should
maintain a hardcoded set:

```python
GENDER_AWARE_LANGUAGES = {
    "Hebrew", "Arabic", "French", "Spanish", "Italian", "Portuguese",
    "German", "Russian", "Polish", "Ukrainian", "Hindi", "Romanian"
}
```

If `TARGET_LANGUAGE` is in this set, run the full diarization + gender detection
pipeline and pass gender to the Claude prompt. If it is not in this set (e.g.
`English`, `Japanese`, `Turkish`), skip diarization and gender detection entirely
— just transcribe and translate, which is faster and saves VRAM.

### `pipeline/transcribe.py`

- Load `WhisperModel` once at startup (singleton), not per request.
- Expose `transcribe(audio_path: Path) -> list[Segment]` where `Segment` is a
  dataclass with `start`, `end`, `text`.
- Settings: `beam_size=5`, `vad_filter=True`, `word_timestamps=False`.
- Model must be loaded lazily on first request and cached for subsequent calls.
- Wrap loading in a threading lock so concurrent requests don't double-load.

### `pipeline/diarize.py`

- Load `pyannote/speaker-diarization-3.1` pipeline once at startup (same singleton
  pattern as transcribe.py).
- Move to CUDA if available.
- Expose `diarize(audio_path: Path) -> Annotation` (returns pyannote Annotation).
- Expose `assign_speaker(segment: Segment, diarization: Annotation) -> str | None`
  which finds the speaker label at the midpoint of the segment.

### `pipeline/gender.py`

- Expose `detect_genders(audio_path: Path, diarization: Annotation) -> dict[str, str]`
  returning `{speaker_id: "male" | "female"}`.
- Load audio with `librosa.load(sr=16000, mono=True)`.
- For each speaker's combined segments, run `librosa.pyin` to extract F0.
- Skip segments shorter than 0.3 s.
- Classify by median F0 vs `GENDER_THRESHOLD_HZ`.
- If no voiced frames detected for a speaker, default to `"male"`.

### `pipeline/translate.py`

- Expose `translate_segment(text: str, gender: str, client: anthropic.Anthropic) -> str`.
- Accepts `target_language: str` and `gender: str | None`.
- If `gender` is provided (i.e. target language is gender-aware), the system prompt must:
    - State the speaker's gender explicitly.
    - Instruct the model to use correct grammatical gender forms throughout
      in the target language (verb conjugation, adjective agreement, imperatives, pronouns).
- If `gender` is `None` (non-gendered language), the prompt simply asks for a clean translation.
- Return only the translated text, no explanation.
- Use `claude-opus-4-6` by default (configurable via `CLAUDE_MODEL`).
- `max_tokens=1024`.

### `server.py`

Build with **FastAPI** + **uvicorn**.

Endpoints:

**`GET /`**
Returns `{"status": "ok", "version": "1.0.0", "model": "<WHISPER_MODEL>"}`.

**`GET /docs`** (auto-provided by FastAPI)

**`POST /asr`**
- Accept `audio_file: UploadFile`, `task: str = "transcribe"`,
  `language: str = "en"`, `output: str = "srt"`, `encode: bool = True`.
- Save uploaded file to a temp directory.
- If `encode=True`, re-encode with ffmpeg to a 16kHz mono WAV first
  (`ffmpeg -i input -ar 16000 -ac 1 output.wav`).
- Use a `asyncio.Semaphore` (size = `CONCURRENT_JOBS`) to limit parallel jobs.
- If `TARGET_LANGUAGE=none`: transcribe only, return as-is.
- If `TARGET_LANGUAGE` is a gender-aware language: run full pipeline
  (transcribe → diarize → gender → translate with gender context).
- If `TARGET_LANGUAGE` is set but not gender-aware: transcribe → translate
  (skip diarization and gender detection).
- Format output as requested (`srt`, `vtt`, `txt`, `json`, `tsv`) and return
  as `PlainTextResponse` with appropriate content-type.
- Clean up temp files after response.
- Log per-request timing at INFO level.

**`GET /status`**
Returns `{"status": "ok", "queue_depth": <int>, "model_loaded": <bool>}`.

### `run.ps1`

PowerShell 7 script that:
1. Prints the local LAN IP (excluding WSL, vEthernet, loopback, APIPA addresses).
2. Prints the full server URL: `http://<LAN_IP>:9000`.
3. Starts the server: `python server.py`.

```powershell
#Requires -Version 7.0
$ip = (Get-NetIPAddress -AddressFamily IPv4 -ErrorAction SilentlyContinue |
       Where-Object { $_.IPAddress -notlike '169.*' -and $_.IPAddress -ne '127.0.0.1' -and
                      $_.InterfaceAlias -notlike '*WSL*' -and $_.InterfaceAlias -notlike '*vEthernet*' -and
                      $_.InterfaceAlias -notlike '*Loopback*' } |
       Select-Object -First 1).IPAddress
Write-Host ""
Write-Host "Server running at: http://${ip}:9000"
Write-Host ""
python server.py
```

## Hardware context

- GPU: NVIDIA RTX 2070 (8 GB VRAM)
- CUDA 13.1, Driver 591.86
- OS: Windows (running natively, not in Docker)

With `large-v3` + `float16`, VRAM usage is ~4.5 GB. Pyannote diarization adds
another ~1–1.5 GB. Total ~6 GB — fits within 8 GB with headroom.
Set `CONCURRENT_JOBS=1` by default.

## Performance requirements

### Never block the async event loop
FastAPI runs on an async event loop. All ML inference (faster-whisper, pyannote,
librosa, Claude API) is blocking CPU/GPU work. Calling these directly inside an
`async def` endpoint will freeze the entire server for the duration of the call —
no other requests can be handled.

Every blocking call must be dispatched to a `ThreadPoolExecutor`:

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

_executor = ThreadPoolExecutor(max_workers=2)

async def run_in_thread(fn, *args):
    loop = asyncio.get_event_loop()
    return await loop.run_in_executor(_executor, fn, *args)
```

Use `run_in_thread` for every ML call inside endpoint handlers.

### Model warm-up at startup
The first inference is always slow: CUDA kernels are JIT-compiled, model weights
are paged into GPU memory. Run a silent dummy pass for both Whisper and pyannote
during the FastAPI `startup` event (before the server accepts requests) so the
first real request is not penalised.

Use a 1-second silent WAV file generated in-memory with numpy/soundfile.

### torch.inference_mode()
Wrap all pyannote and any direct torch calls with `torch.inference_mode()` context
manager. This disables gradient tracking, reduces memory usage, and speeds up
every forward pass. faster-whisper (ctranslate2) handles this internally already.

### Single uvicorn worker
Start uvicorn with `workers=1`. Multiple workers would each load their own copy
of Whisper + pyannote into VRAM — impossible on 8 GB. Single worker with async
concurrency is the correct model here.

In `server.py` at the bottom:

```python
if __name__ == "__main__":
    import uvicorn
    uvicorn.run("server:app", host="0.0.0.0", port=settings.PORT,
                workers=1, log_level="info")
```

### CONCURRENT_JOBS semaphore
The `asyncio.Semaphore` already in the spec gates how many jobs run on the GPU
simultaneously. Keep it at `1` for `large-v3` + pyannote on 8 GB VRAM.
The semaphore must be acquired *before* dispatching to the thread executor.

## Constraints and quality bar

- No Docker. Native Windows Python.
- All ML models (Whisper, pyannote) must be loaded once at startup and reused
  across requests — never reloaded per request.
- Temp audio files must always be cleaned up, even on exception (use `try/finally`).
- The `/asr` endpoint must be compatible enough with `whisper-asr-webservice` that
  Bazarr's Whisper provider works without modification.
- Hebrew text in SRT output must be UTF-8 encoded.
- Write clear log output: model loading, per-request pipeline steps, timing, errors.
